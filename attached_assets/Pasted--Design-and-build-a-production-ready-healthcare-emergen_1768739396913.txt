
Design and build a production-ready healthcare & emergency assistance mobile application named “SaluraCare”.
The app UI, layout, and user flow must strictly follow a modern card-based, minimal medical design similar to the provided reference screens.

================================
DESIGN LANGUAGE & UI STYLE
================================
• Primary colors: Teal & Navy Blue
• Soft gradients, rounded cards, clean spacing
• Medical, calm, trustworthy feel
• Bottom tab navigation (5 tabs)
• Light & Dark mode support
• Icon-driven actions with minimal text
• Fast, one-hand usability

================================
BOTTOM NAVIGATION STRUCTURE
================================
1. Coach (Daily Health)
2. For You (Recommendations)
3. Shop (Medical Store)
4. Reorder (History & Reports)
5. Services (Care Providers)

================================
SCREEN 1: DAILY HEALTH / COACH
================================
Purpose: Daily habit & medicine tracking

UI Components:
• Greeting header with profile icon
• Daily Task toggle (Daily Task / Your Coach)
• Reward card (progress based)
• Drink Water card:
  - Daily goal (liters)
  - One-tap “Done” button
• Medicine Reminder card:
  - Medicine list
  - Dosage & timing
  - Done button per dose
• Progress indicators (1/3, 2/3 etc.)

Features:
• Water reminder notifications
• Medicine reminders with missed-dose alerts
• Streak & reward system
• Daily health summary

================================
SCREEN 2: SHOP — SALURAMART
================================
Purpose: Medical & surgical e-commerce

UI Components:
• Search bar (“Search Medical Supplies”)
• Category chips (Masks, Gloves, Syringes, Dressing, etc.)
• Product cards with:
  - Image
  - Name
  - Quantity
  - Price
  - “Add to Cart” CTA

Product Types:
• Surgical & medical items
• Buy & Rent supported
• Rent plans: daily / weekly / monthly

Features:
• Cart & checkout
• Address & payment
• Order tracking
• Reorder support

================================
SCREEN 3: SERVICES
================================
Purpose: Book healthcare professionals

Service Cards:
• Profile image
• Name & role (Physiotherapist, Home Nurse, Caretaker)
• Experience (years)
• Availability indicator

Supported Services:
• Home nurses
• Caretakers
• Physiotherapists:
  - Orthopedic rehab
  - Post-surgery recovery
  - Neuro physiotherapy
  - Elderly mobility therapy
  - Sports injury rehab

Features:
• Search & filter services
• Book appointment
• Home / clinic visit option
• Ratings & feedback
• Appointment history

================================
EMERGENCY & SOS (GLOBAL)
================================
• Floating SOS button accessible from all screens
• One-tap SOS
• Ambulance request
• Live ambulance tracking
• Emergency contact alerts
• Nearby hospitals on map

================================
LAB TESTS & REPORTS
================================
• Doorstep lab test booking
• Report upload (PDF / image)
• Secure cloud storage
• Download & share reports
• Test history

================================
USER AUTH & PROFILE
================================
• OTP-based login
• Firebase Authentication
• Emergency contacts
• Health details (age, blood group, allergies)

================================
ADMIN & BACKEND
================================
• Firebase + Firestore backend
• Cloud Storage
• Role-based admin panel
• Manage:
  - Users
  - Orders
  - Services
  - Products
  - Healthcare providers

================================
TECH STACK
================================
• Frontend: React Native
• Backend: Firebase
• Notifications: Firebase Cloud Messaging
• Maps: Google Maps API
• Auth: OTP login

================================
DELIVERABLES
================================
• UI screen designs (matching reference UI)
• App architecture
• Folder structure
• Database schema
• API logic
• Admin panel features
• Deployment-ready build

Design and build the app as a real-world healthcare product with focus on speed, simplicity, and emergency-first experience.
